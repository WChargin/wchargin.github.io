Overview

When I was offered the position of student grader, I was disappointed to find that the existing grading system was a jumble of giant, monolithic Bash scripts. The scripts worked, but just barely, and trying to add any kind of new functionality would have been asking for trouble.

To fix this, I designed a modular grading system, based on the premise that any grading process can be broken down into a sequence of discrete steps. For example, the process for grading a Java assignment might be as follows:

  • Copy student's source files to the grading directory.
  • Copy the test driver to the grading directory.
  • Compile the source files with javac.
  • Run the test driver, and make sure that it passes.
  • Deduct points for overly long lines.
  • Deduct points for tab characters.
  • Deduct points for incomplete documentation.
  • Email scores to students.
Most of these steps are not specific to any one assignment, or even any one programming language. By writing each of these steps as its own function, it becomes trivially easy to create assignment specifications that can be automatically executed.

Features

  • Support for multiple grading periods, and penalties for late submissions or repeated submissions (partial credit). This includes a tool to extract each student's best score and export the results as CSV for easy input into a gradebook.

  • Integration with cron via a configurable schedule file that lists upcoming assignments, grading periods, and parameters.

  • Language bindings for steps written in Bash and Python, with the possibility of easy expansion to other languages.

  • Wide variety of built-in steps, including:

    • simple file manipulation scripts (e.g., “copy files from student directory to testing directory”);
    • custom Javadoc doclet to automatically check and score documentation;
    • style checking tools for long lines, tab characters, etc.;
    • Java test driver runner;
    • utility to mail scores and deduction explanations to students.

  • And, of course, comprehensive documentation for instructors and new student graders.

Deployment

My system is in use by Computer Science 102 classes at Cal Poly.
